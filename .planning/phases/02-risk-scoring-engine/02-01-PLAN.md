**Important clarification on the threshold semantics (planner read both CONTEXT.md and ROADMAP.md):**
- The risk_score is a "risk magnitude" — HIGHER = MORE risk (`>= 75 → CRITICAL`, `< 25 → LOW`) per CONTEXT.md D-06.
- Each component score is a "risk magnitude" too: D-01 says `(1 - attendance_days/14) * 100` — 0/14 days → 100 risk, 14/14 → 0 risk. Same shape for D-02 (practice) and D-04 (notes); D-03 is binary 100/0 with 50 neutral fallback.
- Worst student (all zero engagement, no note): all four components = 100 → `risk_score = 100 * (0.35 + 0.30 + 0.20 + 0.15) = 100` → CRITICAL (`>= 75`). Matches ROADMAP Success Criterion #2.
- Perfect student (full engagement, recent note, improving): all four components = 0 → `risk_score = 0` → LOW (`< 25`). Matches ROADMAP Success Criterion #3.
- No contradiction exists between CONTEXT.md and ROADMAP.md. Both are consistent with the "higher score = more at-risk" interpretation. (The planning-guidance memo flagged a contradiction that does not actually exist once the component formulas are evaluated — components are inverted from raw activity, so worst raw inputs produce the highest score.)
